# System Design & Operating Model

This document describes how **research-derived-experiments** works end-to-end:
major components, runtime flow, configuration, dependencies, environment setup,
deployment, assumptions, limitations, and what the system can and cannot do.

For a package-by-package catalog (experiments, scripts, tests, docs), see
[module_explainer.md](module_explainer.md). Keep both files updated when the
codebase changes in a meaningful way (see root `AGENTS.md`).

---

## 1. What this repository is

This is a **public-safe research program repository**, not a single deployable
application. It tracks experiments, papers, provenance, and a small set of
product/atlas surfaces around a recurring thesis:

> Adaptive systems keep rediscovering geometric language because geometry is the
> portable language of constraints.

The working scientific question is whether independently developed systems of
thought and computation converge on geometric descriptions of meaning, agency,
and intelligence — and when that geometry is a passive representation versus an
active, self-maintaining attractor regime.

Operationally, the repo is designed for **AI-agent-generated science that remains
observable**: attribution, one-command regeneration where possible, committed
summaries instead of raw dumps, and discovery-regime audits that preserve
accepted *and* rejected alternatives.

Human author & director: **Jawaun Brown**. Experiment code, sweeps, and paper
drafts are generated by AI coding agents under the author's direction and review.

## 2. High-level architecture

```text
┌──────────────────────────────────────────────────────────────────────────┐
│                         Research program (this repo)                      │
│                                                                          │
│  notes/ ── thesis & synthesis                                            │
│  experiments/<name>/ ── harnesses, Modal sweeps, results, PROVENANCE     │
│  papers/<name>/ ── paper.md (+ figures/PDFs)                             │
│  formal/ontology-hs/ ── Haskell typed gate for Arc 2B                    │
│  scripts/ ── quality, provenance, PDF/figure builders, summarizers       │
│  tests/ ── unittest suite for core harnesses                             │
│  docs/ ── handoffs, verification, benchmark schema, this design doc      │
│  references/ ── public source list; local-only full texts (gitignored)   │
└───────────────────────────────┬──────────────────────────────────────────┘
                                │
          ┌─────────────────────┼─────────────────────┐
          ▼                     ▼                     ▼
   Modal (GPU/CPU)        Local CPU pilots      Railway (static sites)
   Doppler secrets        uvx Python 3.12       atlas + inquiry landing
          │                     │                     │
          └────────► artifacts/ (gitignored raw JSON) │
                     summarized into experiments/*/results/*.md
                     PDFs → papers/pdf/ + sites/
```

There is **no monolithic installable Python package**. Experiments are
directories with runnable modules; shared tooling lives under `scripts/`
(especially `paperkit.py`). Dependencies are pulled **ephemerally** via `uvx`
or Modal images, not from a root lockfile.

Two adjacent product surfaces live in-tree but are operationally separate:

| Surface | Path | Role |
|---|---|---|
| Research Mechanism Atlas | `sites/reafference_attribution/` | Public static site serving papers, verification index, mechanism visuals |
| Inquiry Black Box (landing) | `sites/inquiry_black_box/` | Marketing/landing for the local-first capture product |
| Inquiry Black Box (app) | `apps/inquiry-black-box/` | Electron + Chrome extension monorepo (Bun/TypeScript) |
| Coherence testbench | `coherence-testbench/` | Phase-0 EEG/eyetrack GO/KILL bench (own `pyproject.toml`, Modal jobs) |

## 3. Major components and how they interact

### 3.1 Experiment packages (`experiments/`)

Each experiment directory is a self-contained research unit. Typical contents:

| Artifact | Role |
|---|---|
| `README.md` / hypothesis notes | Claim, gates, controls |
| `*.py` harness modules | Local CPU or shared core logic |
| `modal_*.py` | Modal app entrypoints (`@app.function` + `@app.local_entrypoint`) |
| `results/*.md` | **Committed** human-readable reports (headline numbers, gates, caveats) |
| `PROVENANCE.md` | Attribution, run command, seed, verification signals, regen path |
| `BENCHMARK_CARD.md` | Present on hardened public benchmark suites (e.g. Suite C, long-horizon) |

Raw run outputs go to **gitignored** `artifacts/<experiment>/`. Only summarized
reports and intentionally public tables/PDFs are committed.

There are ~49 experiment directories on disk; ~44 are registered in
`docs/verification.json` (the machine-readable provenance manifest).

### 3.2 Papers (`papers/`)

One directory per paper or synthesis topic, usually mirroring an experiment name.
Common layout: `paper.md`, optional `preregistration.md` / `runbook.md`, figures,
and rendered PDFs under `papers/pdf/` or the paper directory.

PDF builders under `scripts/build_*_pdf.py` read **committed numbers** (result
reports / small CSV snapshots), not live Modal logs. Shared rendering lives in
`scripts/paperkit.py` (reportlab + matplotlib; no LaTeX required).

### 3.3 Scripts (`scripts/`)

Operational spine of the repo:

| Concern | Key scripts |
|---|---|
| Quality gate | `run_quality_checks.py`, `publication_guard.py` |
| Provenance | `gen_provenance.py`, `regen.py` |
| Secrets hygiene | `env_probe.py` |
| Publication | `build_*_pdf.py`, `make_*_figures.py`, `paperkit.py` |
| Summarization | `summarize_*.py` |
| Remote paper tasks | `modal_*_paper_tasks.py` |

### 3.4 Verification & provenance

```text
scripts/gen_provenance.py
    → experiments/<name>/PROVENANCE.md
    → docs/verification.json
    → docs/verification.md
    → sites/reafference_attribution/verification.json
```

`scripts/regen.py` is the one-command reproducer/dispatcher:

```bash
python scripts/regen.py list
python scripts/regen.py <experiment_name>
```

Modes:

| Mode | Behavior |
|---|---|
| `local-cpu` | Runs deterministic CPU command chains (e.g. `grid_cell_weakness`, `weakness_temporal`) |
| `pdf` | Rebuilds PDF from committed numbers (e.g. `symbolic_weakness`) |
| `documented-command` | Prints the Modal/Doppler dispatch from `verification.json` |

### 3.5 Formal gate (`formal/ontology-hs/`)

Haskell Cabal project that validates typed computational-body motifs for Arc 2B.
Python calls it via `experiments/viable_computational_bodies/haskell_gate.py`.

```bash
cd formal/ontology-hs && cabal test all && cabal run ontology-check
```

### 3.6 Public sites & CI deploy

On push to `main`, `.github/workflows/railway-deploy.yml` deploys:

1. **Reafference atlas** (repo root / atlas service)
2. **Inquiry Black Box landing** (`sites/inquiry_black_box`)

Requires GitHub secret `RAILWAY_TOKEN`. Native Railway GitHub autodeploy has
historically been unreliable; Actions is the supported path. See
[railway-autodeploy.md](railway-autodeploy.md).

There is **no** GitHub Actions workflow for tests/lint; quality is local via
`scripts/run_quality_checks.py`.

## 4. Research arcs (scientific operating model)

The program is organized as overlapping arcs, not a single pipeline:

| Arc / track | Core packages | Claim shape |
|---|---|---|
| **Weakness → OOD** | `weakness_vs_simplicity`, `symbolic_weakness`, `rotation_weakness`, `learned_symmetry`, `grid_cell_weakness`, `structure_compatible_generalization` | Symmetry-/structure-compatible hypothesis weakness predicts OOD where simplicity/MDL/flatness do not |
| **Concept / activation geometry** | `concept_geometry`, `activation_geometry`, `paraphrase_weakness`, `passive_to_active` | Bridge geometry in embeddings and hidden states; when it becomes causally load-bearing |
| **Maintained concern / metric stack** | Papers 5–25 family under `experiments/*` (valence, homeostatic, planning, first-order self, world responds, …) | Concern/ΔE geometry as control substrate; self/world attribution; re-engagement |
| **Arc 2A Concerned syntax** | `concerned_syntax` | Causal constituency + concern-gated intervention invention |
| **Arc 2B Viable bodies** | `viable_computational_bodies` + `formal/ontology-hs` | Typed architecture search under viability + formal + syntax gates |
| **Causally grounded agents benchmark** | `world_responds` (Suite C), `long_horizon_bottleneck` (Suite D/E), umbrella docs | Behavior + structure-specific gate; proxy-resistant agent evaluation |
| **External contact / phases 4–6** | `external_contact`, `phase4_metaphysics`, `phase5_external_validity`, `phase6_real_model_validation` | Transport claims beyond in-lab harnesses toward real models |

Process wrapper: discovery-regime audit (pre-register gates, preserve rejects,
distinguish retrieval/search/discovery). Ledger: root `TODO.md`. Audit cards:
[discovery_regime_audit.md](discovery_regime_audit.md).

**Minimum pass rule** for the agent benchmark family: behavior plus at least one
structure-specific gate (see [causally_grounded_agents_benchmark.md](causally_grounded_agents_benchmark.md)).

## 5. Runtime flow

### 5.1 Standard experiment loop

```text
1. Pre-register claim + gates (paper note / plan / audit card)
2. Local CPU pilot (small trials, fixture provider where available)
3. Optional Modal sweep (Doppler → uvx → modal run)
4. Raw JSON/JSONL → artifacts/<name>/   (gitignored)
5. summarize_*.py or experiment summarize → experiments/<name>/results/*.md
6. Optional: make_*_figures.py / build_*_pdf.py → papers/pdf/ + site copies
7. python scripts/gen_provenance.py
8. python scripts/run_quality_checks.py
9. PR → merge main → Railway deploy (sites)
```

### 5.2 Modal dispatch pattern

House pattern for GPU/large sweeps:

```bash
doppler --scope /Users/jawaun/superoptimizers run -- \
  uvx --python 3.12 --from modal [--with numpy|torch|...] modal run \
  experiments/<name>/modal_<sweep>.py \
  --flags \
  --out artifacts/<name>/...
```

Inside Modal scripts: `@app.function` workers + `@app.local_entrypoint` that
`.map()` over seeds/shards. Secrets come from Doppler or a local Modal profile
(`.modal.toml` is gitignored).

### 5.3 Local CPU / fixture paths

Examples that do **not** require Modal for a meaningful smoke:

```bash
# Weakness vs simplicity pilot
python3 experiments/weakness_vs_simplicity/experiment.py \
  --trials 500 --seed 7 --out artifacts/weakness_vs_simplicity/pilot.json

# Long-horizon public fixture smoke
python3 -m experiments.long_horizon_bottleneck.eval \
  --provider fixture --models fixture --suite prompt_family \
  --seeds 1 --episodes-per-cell 1 \
  --out artifacts/long_horizon_bottleneck/fixture_public_smoke_summary.json \
  --jsonl artifacts/long_horizon_bottleneck/fixture_public_smoke_rows.jsonl

# One-command regen (where supported)
python scripts/regen.py grid_cell_weakness
python scripts/regen.py weakness_temporal
```

### 5.4 Quality gate pipeline

`scripts/run_quality_checks.py` (Python 3.12 via `uvx`):

1. `unittest discover -s tests` (with torch, numpy, scikit-learn, pytest)
2. `compileall` on `scripts`, `experiments`, `tests`
3. `publication_guard.py` (public-repo safety)
4. `ruff check .`
5. `ty check` on selected trees (many Modal/legacy paths excluded in `pyproject.toml`)

## 6. Configuration & dependencies

### 6.1 Language / tooling

| Item | Requirement |
|---|---|
| Python | `>=3.12` (`pyproject.toml`) |
| Package manager for checks | `uvx` (ephemeral deps; no root `requirements.txt`) |
| Lint | Ruff (relaxed ignores for historical Modal notebook-style scripts) |
| Types | `ty` with broad excludes for Modal scripts and `scripts/**` |
| Haskell (Arc 2B formal gate) | Cabal / GHC for `formal/ontology-hs` |
| Inquiry app | Bun + TypeScript (`apps/inquiry-black-box`) |
| Sites | Node (`server.js`, `PORT`, default 3000) |

### 6.2 Environment variables

Probe without leaking values:

```bash
python3 scripts/env_probe.py
doppler run -- python3 scripts/env_probe.py
```

| Group | Variables |
|---|---|
| Model APIs (common) | `OPENAI_API_KEY`, `ANTHROPIC_API_KEY` |
| Optional APIs | `GOOGLE_API_KEY`, `GEMINI_API_KEY`, `HUGGINGFACE_TOKEN` / `HF_TOKEN` |
| Modal | `MODAL_TOKEN_ID`, `MODAL_TOKEN_SECRET`, `MODAL_WORKSPACE`, `MODAL_ENVIRONMENT`, `MODAL_DEFAULT_GPU` |
| Deploy | `RAILWAY_TOKEN` (GitHub Actions); `PORT` (Railway runtime) |

**Do not commit secrets.** `.env`, `.doppler/`, `.modal.toml` are gitignored.

### 6.3 Artifact & data conventions

| Location | Tracked? | Contents |
|---|---|---|
| `artifacts/` | No | Raw JSON/JSONL, local PDF build scratch |
| `experiments/<name>/results/` | Yes | Summarized result reports |
| `papers/`, `papers/pdf/` | Yes | Paper sources and shareable PDFs |
| `data/` | No (except `data/paper_b/`) | Large/raw data; Paper B CSV snapshots are the committed exception |
| `references/papers|text|html/` | No | Full-text citation archive |
| `tmp/` | No | Scratch (literature audits, etc.) |
| `docs/verification.json` | Yes | Machine-readable provenance manifest |

Public policy: publish notes, code, summarized reports, audit cards, rejected
alternatives; keep raw model dumps, secrets, and full-text PDFs local unless
intentionally summarized. Enforced by `publication_guard.py`.

## 7. Environment setup (contributor quickstart)

1. Clone the repo; use Python 3.12+ (system 3.9 is insufficient for the quality gate).
2. Install `uv` / `uvx` for ephemeral scientific deps.
3. Optional: configure Doppler scope and Modal credentials for GPU sweeps.
4. Verify secrets presence: `python3 scripts/env_probe.py`.
5. Run quality checks: `python3 scripts/run_quality_checks.py`.
6. List reproducible experiments: `python scripts/regen.py list`.
7. For Modal handoff context: [next_agent_modal_handoff.md](next_agent_modal_handoff.md).
8. For public atlas deploy: [railway-autodeploy.md](railway-autodeploy.md).

Haskell formal gate (only if touching Arc 2B typed ontology):

```bash
cd formal/ontology-hs && cabal test all && cabal run ontology-check
```

Inquiry Black Box app (separate stack):

```bash
cd apps/inquiry-black-box && bun install && bun run test && bun run build:prototype
```

Coherence testbench (separate stack):

```bash
cd coherence-testbench && python3 scripts/run_phase0.py --smoke
```

## 9. What the system can do

- Run and document a large portfolio of **proxy-resistant** and **structure-gated**
  experiments (weakness/OOD, concern/control, concerned syntax, viable bodies,
  Suite C / long-horizon agent benchmarks).
- Keep the public GitHub surface **safe by construction** (summaries in, raw dumps out).
- Regenerate provenance cards and a verification index for agent-generated science.
- Reproduce a subset of experiments on CPU with one command (`regen.py`).
- Dispatch documented Modal commands for the rest.
- Build shareable PDFs/figures from committed evidence.
- Deploy a public research atlas and Inquiry landing site via Railway.
- Host adjacent product/testbench code (`apps/`, `coherence-testbench/`) without
  forcing them into the Python experiment quality gate.

---

## 10. What the system cannot do (limitations)

- **Not a single service.** There is no unified API or orchestration daemon for all experiments.
- **Most experiments are not one-click local.** `regen.py` prints Modal commands for the majority; you need secrets and Modal access.
- **No CI test/lint on push.** Quality depends on contributors running `run_quality_checks.py` before merge.
- **Weak static checking on Modal/legacy scripts.** `ty` and Ruff intentionally exclude or relax large historical surfaces.
- **No root dependency lockfile.** Reproducibility of the *environment* relies on uvx pins at call sites and Modal images, not a single `uv.lock`.
- **Machine-specific paths** appear in docs/handoffs (Doppler scope `/Users/jawaun/superoptimizers`, local archive folders). Adapt for other machines.
- **Result fidelity depends on summarization discipline.** Gitignored JSON is the raw truth of a sweep; committed Markdown is the public contract — they can drift if summarizers are skipped.
- **Coherence-testbench and Inquiry app** have their own runtimes and are not fully wired into the root Python quality gate.
- **Scientific claims are gate-bound.** Passing a fixture smoke or a single suite does not imply the whole program thesis is settled; see audit cards and paper claim boundaries.

---

## 11. Assumptions

1. Contributors treat `experiments/*/results/*.md` and provenance as the public scientific record.
2. Large sweeps are pre-registered; negatives and rejects are preserved, not deleted.
3. Secrets live in Doppler / Modal / GitHub Actions — never in git.
4. PDF builders consume committed summaries/CSVs, not private local archives, for public rebuilds.
5. Attribution remains explicit: human director + AI-generated implementation under review.
6. Python 3.12+ is available for the quality gate even if the host default is older.
