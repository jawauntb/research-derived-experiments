## Question

Does symmetry-compatible-hypothesis weakness (the equivariance count of a learned function under the task's true group) predict OOD generalization on the Pythia model suite — i.e., does the lab's weakness paper headline (weakness ρ ≈ +0.81 with OOD on its own MLP sweep, beating loss / scale / L₂ / sharpness) transfer to an external pretrained model family?

Per Pythia size: load checkpoint once, freeze weights, take the last-layer hidden state of the last token as a fixed feature for each integer a ∈ {0..n-1}, train a small linear head on a strict-subset partial orbit (50% of inputs shown), extract the argmax function table over the full domain, score weakness_oracle under Z_n + the wrong-group null, compute classical predictors (head loss, eval NLL on OOD, Pythia param count, Pythia L₂, Hutchinson sharpness on the head loss), then regress all predictors against held-out OOD accuracy.

## Result — degenerate, not a clean falsification

**Every single one of the 27 cells produced ood_accuracy = 0.000.** The head trains to 100 % training accuracy on its partial orbit but cannot extrapolate the modular-shift function to held-out inputs at any (size, n, seed). The Spearman ρ values therefore collapse:

| Statistic | Value |
|---|---:|
| `n_cells` (valid) | 27 |
| `rho(weakness_oracle_norm, OOD)` | **0.000** |
| `rho(weakness_wrong_group_norm, OOD)` | 0.000 |
| `rho(head_train_loss, OOD)` | 0.000 |
| `rho(ood_nll, OOD)` | 0.000 |
| `rho(pythia_param_count, OOD)` | 0.000 |
| `rho(pythia_l2, OOD)` | 0.000 |
| `rho(head_sharpness_proxy, OOD)` | 0.000 |
| `best_classical_abs_rho` | 0.000 |
| `P1_pass` | **false** (literal threshold) |
| `P1_hard_kill` | **true** (mechanically: ρ < 0.3 ⇒ true) |

Distribution of `weakness_oracle_norm` across the 27 cells: 9 × 0.077 (n=13), 9 × 0.059 (n=17), 9 × 0.043 (n=23) — equal to 1/n at each n, i.e. only the identity element of Z_n maps the function table to itself. Distribution of `ood_accuracy`: 27 × 0.0.

This is **not** the "weakness signal failed to predict OOD" result the pre-registration's hard-kill clause describes. The Pythia linear probe (frozen features + a single Linear layer) never produced a non-trivial OOD prediction in *any* cell, so there was no within-sweep variation to correlate against. The test mechanically reports `P1_hard_kill = true`, but the substantive interpretation is **the experimental setup degenerated** — linear probing on frozen Pythia hidden states is too constrained to elicit a learned modular-arithmetic function at all.

### Why it degenerated (honest diagnosis)

1. **Linear probe ≠ fine-tune.** The runbook offered "linear / LoRA head"; this run used the strict linear variant. The weakness paper's headline came from MLPs that *learn the task* — partial-orbit supervision shapes the internal representations. A linear head on frozen Pythia features can only re-weight features Pythia already encodes, and Pythia's representations of bare digit strings (`"5"`, `"12"`) almost certainly do not encode `+ offset mod n` linearly. With the digit-string features it has, the linear head memorizes the train half-orbit perfectly (head_train_accuracy = 1.0 in every cell) and produces noise on OOD inputs.
2. **The function table is therefore dominated by the head's noise pattern, not Pythia's "function".** That makes the equivariance-count *of the linear-probe argmax* an artifact of the head initialization, not a property of Pythia's learned function. The weakness statistic is well-defined but uninformative.
3. **Identical OOD = 0.0 across all sizes** is the strongest signal here: scale (70m → 410m, 5.9× param count) does *not* help in this setup. So even the parameter-count baseline (which P1 was supposed to beat) is dead.

### What this run CAN and CANNOT conclude

**Can conclude:**

- The literal P1 threshold (ρ ≥ 0.5 for weakness vs OOD, beating classical predictors by 0.25 in |ρ|, wrong-group |ρ| ≤ 0.15) is **not evaluable** from this Tier-B variant. All ρ values are 0.0 because the dependent variable is constant — Spearman ρ on a constant column is mechanically zero, not informative.
- Linear probing on frozen Pythia digit-string features is a degenerate operationalization of "learned function weakness". The runbook's linear / LoRA option needs to be taken with LoRA (or full fine-tune), not linear-only.
- Therefore **the program's first external-load-bearing field-claim candidate (P1) is NOT validated here; nor is it cleanly falsified**. It is *unsettled*, awaiting a non-degenerate Tier-B.

**Cannot conclude:**

- "Weakness does not transfer to Pythia" — that requires a setup where Pythia *actually learns* the modular-shift function (some cells getting non-zero OOD) and we observe the weakness signal failing to predict OOD across cells.
- "The lab's weakness paper headline depends on its hand-built MLPs" — same; the negative would need to come from a setting where another model family *can* learn the task with some variance in OOD performance to correlate against.

## Allowed claim

**Inconclusive Tier-B / methodology degeneration.** The literal P1 threshold remains unsettled. The cleanest write-up of this run is: "Linear probing on frozen Pythia hidden states fails to elicit any modular-arithmetic generalization across pythia-70m / 160m / 410m at n ∈ {13, 17, 23}, train_frac = 0.5, head_epochs = 400. The literal P1 threshold cannot be evaluated under this configuration."

## Discovery-Regime Audit

- **Old regime:** P1's literal threshold (Spearman ρ ≥ +0.5 for weakness vs OOD on Pythia, beating loss / scale / L₂ / sharpness by ≥ 0.25 in |ρ|) was the program's strongest "external field-claim" candidate. The pre-registration was conservative (linear / LoRA head; not full fine-tune) precisely to keep Tier-B tractable. The Tier-A surrogate (orbit-coverage proxy, transcribed numbers) was the explicit fallback.
- **Transition:** Tier-B was actually attempted on Modal with the linear-probe option. 27 cells (3 sizes × 3 ns × 3 seeds) all completed cleanly. The setup is a faithful realization of the runbook §P1 "linear head" path.
- **Transported evidence:** the lab's `equiv_count` weakness logic from `experiments/symbolic_weakness/selectors.py` was re-implemented inline (the Modal worker is self-contained) and applied verbatim to the Pythia-derived function tables. The classical-predictor stack (loss, eval NLL, param count, L₂, Hutchinson sharpness) is computed on the same artifacts.
- **Rejected alternatives:** "linear probe on frozen Pythia is sufficient to elicit modular-shift generalization at any size up to 410m" is rejected by the all-zero OOD column. "Scale alone helps on this task" is also rejected (70m, 160m, 410m all 0.0 OOD).
- **Residual finding:** the linear-probe Tier-B operationalization is degenerate; the literal P1 threshold is unevaluable in this configuration.
- **Honest weakness:** This is *not* an external falsification of the weakness paper's claim; it is a tooling result. The substantive Tier-B for P1 needs LoRA fine-tuning at minimum (or full fine-tune), which is a meaningfully larger Modal run.
- **Allowed claim:** **inconclusive Tier-B / methodology degeneration.** P1 is unsettled.
- **Next operation:** re-run P1 Tier-B with LoRA (e.g. rank-8 LoRA on the attention + MLP projections of the last 2 layers, plus a Linear classification head over n classes) so the model can actually adapt its representations to the modular-shift task. Budget conservatively: the same 27-cell sweep, ~5× the per-cell cost vs the linear probe.
