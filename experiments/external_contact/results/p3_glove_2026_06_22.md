## Question

Does the lab's authored concept-geometry structure survive contact with an
external embedding family the lab did not build? Prediction P3 asks whether the
24-concept geometry from `experiments/concept_geometry/` remains visible in
public GloVe vectors after the pre-registered All-but-the-Top anisotropy
correction.

## Commands

```bash
python3 -m experiments.external_contact.p3_glove_probe --self-test
python3 scripts/fetch_glove_subset.py --dims 300 100 --run
```

The fetch helper downloaded `glove.6B.zip`, extracted only the 300d and 100d
tables into `tmp/glove/`, and vendored only the needed word rows into ignored
subset files:

- `experiments/external_contact/p3_glove_subset_300d.txt`
- `experiments/external_contact/p3_glove_subset_100d.txt`

Subset coverage was 397/399 needed word types for both dimensionalities. Missing
tokens: `autopoiesis`, `underdetermining`. This dropped the `autopoiesis`
concept entirely, so the real run scored 23 concepts rather than 24.

The synthetic self-test passed. It validates the math only; it is not scientific
evidence.

## Result

All three frozen P3 gates pass after centering.

| Gate | Pre-registered threshold | Result | Verdict |
|---|---:|---:|---|
| P3a centered within-across margin | >= 0.10 | 0.106 | PASS |
| P3a clustering NMI vs authored categories | >= 0.25 | 0.531 | PASS |
| P3b centered paraphrase gap | >= 0.15 | 0.252 | PASS |
| P3c cross-model RSA, GloVe-300d vs GloVe-100d | >= 0.60 | 0.747 | PASS |

Raw-vs-centered comparison:

| Metric | Raw | Centered |
|---|---:|---:|
| P3a within-across margin | 0.061 | 0.106 |
| P3b paraphrase gap | 0.072 | 0.252 |

The strongest old-regime shortcut did not carry the result. Raw cosine was below
the pre-registered pass thresholds for both P3a and P3b; the signal strengthened
after All-but-the-Top centering rather than vanishing under it.

## Interpretation

This is the program's first resolved external-contact episode: the category-block
geometry, paraphrase-vs-wrong-orbit gap, and held-out cross-model transfer gate
all survive on a public embedding family the lab did not build.

The allowed claim is **mechanism -> regime transition**, not a field claim.
Static word vectors are a limited substrate, P3c uses two dimensionalities within
one embedding family rather than three independent families, and one concept was
missing from GloVe. Still, the clean result does answer the director's core
critique for this pillar: the concept-geometry signal is not only an OpenAI API
or self-built benchmark artifact.

## Discovery-Regime Audit

- **Old regime:** concept geometry had OpenAI embedding and internal activation
  evidence, with hand-authored concept sets, paraphrase probes, and centered
  similarity controls.
- **Transition:** the same frozen concept/paraphrase artifacts were tested on
  public GloVe vectors, an external non-LLM substrate the lab did not build.
- **Transported evidence:** P3a/P3b/P3c thresholds, All-but-the-Top centering,
  wrong-orbit paraphrase control, and cross-model RSA were preserved from the
  pre-registration.
- **Rejected alternatives:** raw anisotropic cosine did not pass and therefore
  cannot explain the result.
- **Residual finding:** external GloVe preserves the authored category structure
  and paraphrase geometry after centering; cross-dimensional relational geometry
  is also stable.
- **Readiness:** all P3 gates pass; caveats are 23/24 concept coverage and only
  one external embedding family.
- **Allowed claim:** regime transition / external load-bearing diagnostic.
- **Next operation:** add a third external embedding family such as fastText or
  MiniLM, then move to P1/P2 so external load-bearing is tested beyond static
  lexical geometry.
