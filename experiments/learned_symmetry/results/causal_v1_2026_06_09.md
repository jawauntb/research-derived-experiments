# Causal Validation of Learned-Group Augmentation

Date: 2026-06-09

## Question

The prior `modal_sweep_v1` result showed that learned-group weakness
*correlates* with OOD generalization (Pearson r = +0.66 vs oracle r =
+0.74). The natural reviewer objection: is the learned group actually
useful, or just a passive predictor of which models will already
generalize? This sweep addresses that with a paired-comparison design:
for each base config, retrain the SAME model under four augmentation
regimes and measure per-model OOD lift.

## Setup

64 base configs × 4 augmentation regimes = 256 trainings, run on
8 Modal shards × 8 base configs per shard. Each base config is a random
combination of architecture (CNN/MLP) × hidden width × depth × init
scale × optimizer × learning rate × weight decay. For each base config
we train four models with everything held constant except the
augmentation regime:

1. `none` — no augmentation; train on the biased 3-of-8-rotations prefix.
2. `oracle_aug` — augment with rotations from the true Z_8 group.
3. `learned_aug` — augment with rotations from the data-inferred group
   (no oracle access; same procedure as `modal_sweep_v1`).
4. `random_aug` — augment with the same number of randomly-chosen
   rotations from the 24-angle candidate set.

Manifest:

- Command: `doppler --scope /Users/jawaun/superoptimizers run -- uvx --python 3.12 --from modal modal run experiments/learned_symmetry/modal_causal_validation.py --n-shards 8 --n-base-per-shard 8 --epochs 250 --base-seed 20260609 --out artifacts/learned_symmetry/causal_v1.json`
- Total trainings: 256
- Paired causal units: 64

## Gate

Acceptance:

- `learned_aug` mean per-model OOD lift ≥ 80% of `oracle_aug`'s lift.
- `learned_aug` mean per-model OOD lift > `random_aug`'s by ≥ 4 pp.
- Paired `learned − oracle` delta within ±10 pp (one standard deviation).

Withheld:

- This does not yet validate causally on larger images or natural data.
- The random-rotation control is the soft control from §3.4 of the v1
  paper; we additionally expect (but did not run here) a hard
  pixel-permutation control to lie further below.

## Result

### Mean OOD by regime (n=64 per regime)

| Regime | Mean OOD | Stdev |
| --- | ---: | ---: |
| `none` | 0.268 | 0.069 |
| **`learned_aug`** (no oracle) | **0.783** | 0.273 |
| `oracle_aug` (with oracle) | 0.836 | 0.266 |
| `random_aug` | 0.713 | 0.242 |

### Per-model OOD lift vs `none` (paired)

| Regime | Mean lift | Stdev |
| --- | ---: | ---: |
| `oracle_aug` | **+0.568** | 0.259 |
| **`learned_aug`** | **+0.515** | 0.269 |
| `random_aug` | +0.444 | 0.238 |

### Paired regime comparisons (same model, different aug)

| Comparison | Mean Δ | Stdev |
| --- | ---: | ---: |
| `learned_aug` − `oracle_aug` | **−0.053** | 0.078 |
| `learned_aug` − `random_aug` | **+0.070** | 0.079 |

### Group recovery

- Mean learned-group size: 10.31 angles (range 6–16, out of 24 candidates)
- Same procedure as `modal_sweep_v1`: cosine self-consistency with
  threshold 0.5 on raw pixel features.

## Headline

**Training with the *data-inferred* group as augmentation lifts OOD by
+51.5 percentage points vs no augmentation** — within 5.3 pp of the
oracle's +56.8 pp lift, and 7.0 pp above the random-rotation control.
The data alone is sufficient to recover the augmentation set that
produces 90% of the oracle's causal OOD lift.

## Audit

Accepted:

- `learned_aug` lift / `oracle_aug` lift = 0.515 / 0.568 = **90.7%**.
  Passes the 80% acceptance threshold by a wide margin.
- `learned_aug − random_aug = +7.0 pp`. Passes the ≥4 pp acceptance
  threshold.
- Paired `learned − oracle = −5.3 pp` (stdev 7.8 pp). Within the ±10 pp
  acceptance band by less than one standard deviation.

Rejected/withheld:

- The random-rotation control still produces a substantial OOD lift
  (+44.4 pp), reproducing the §3.4 finding that dense candidate grids
  partially overlap Z_8 by chance. The signal is in the *gap* between
  learned and random, not the absolute level of random.
- This is single-domain causal validation (rotated-stroke vision).
  A second domain (paraphrase substitutions on Pythia-70M) is run
  in parallel under `experiments/paraphrase_weakness/modal_learned_substitution.py`.

Residual content:

- The random-rotation control's +44.4 pp lift is interesting in its
  own right: even rotation-flavored augmentation drawn from a dense
  candidate grid produces real generalization. This suggests that
  *some* rotation-equivariance prior is the key, and the data-inferred
  procedure simply finds the right one without overspecifying.
- The 5.3 pp gap between learned and oracle is small but real. Two
  likely sources: (i) near-identity false positives in the learned
  group (15°, 345°) waste a small fraction of the augmentation budget;
  (ii) the learned group occasionally misses a Z_8 element under noise.
